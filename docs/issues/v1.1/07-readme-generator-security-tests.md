# Issue: README Generator Security Tests

## Title
Implement Security & Sanitization Tests for README Generator

## Problem
When generating a workspace README, the engine pulls data from local files. If the user accidentally pastes an LLM API key or sensitive data into a tracked Markdown file, the generated README must absolutely not expose this when pushed to GitHub.

## Goal
Implement a security layer that sanitizes all output generated by `alo readme` before writing it to disk.

## Scope
- Implement a secret-scanning filter in `readme_service.py`.
- Define rigorous unit tests to verify API keys and environment variables are stripped.
- Test SVG generation for accidental credential interpolation.

## Non-goals
- Full filesystem antivirus scanning. We only scan the strings generated by ALO's own templating engine.

## Acceptance Criteria
- Mock payloads containing API keys (e.g. `sk-test...`, `OPENAI_API_KEY=`) are successfully scrubbed or blocked from writing to `README.md`.
- No generated file can stage unknown private configurations to Git.

## Test Requirements
- Parameterized unit tests injecting fake tokens and verifying they are caught.

## Security Considerations
- This is a critical security guardrail. Tests must fail the build if sanitization is bypassed.

## Labels Suggestion
`security`, `testing`, `v1.1.0`
